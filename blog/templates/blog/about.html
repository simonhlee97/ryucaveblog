SimonPost.com, the code

(If you're not a coder/developer, you will find the rest of this page quite boring...)

TL;DR This blog was built with Django, and some vanilla CSS.

The previous iteration of my blog was built with Eleventy, a static site generator, and was hosted on Netlify. I really liked using Eleventy. But after learning some Django, I decided on re-building my blog from scratch to take advantage of Django's built-in frameworks and features. Here's a quick rundown of what I gained with Django.

With Eleventy, I was writing posts in local Markdown files, and then pushing my commits to GitHub (deployed on Netlify). It was a pretty nice workflow.

However, with Django's Auth framework, I don't have to have my MacBook to write drafts or publish posts. I can log in from any computer and write drafts or publish posts. I didn't want to spend a lot of time building an auth system for my Eleventy blog.

With Django, adding features like RSS feeds, pagination, search, and share-by-email were all quite easy to learn and implement. If you're a developer/blogger, and you have not tried out Django, I highly recommend giving it a try. The more I learn about Django, the more I like this framework. There are many other features I have yet to discover or use. And last but not least, Django has great docs and a fantastic community.
